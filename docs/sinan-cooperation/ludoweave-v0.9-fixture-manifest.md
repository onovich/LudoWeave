## Review Notes

- The manifest references existing source and test fixtures only; it does not create a real Sinan adapter.
- Commands use repository scripts and targeted Vitest/Playwright entry points. Final acceptance still requires the complete v0.9 validation matrix.
- Fixture ownership is intentionally split: LudoWeave owns review artifacts and generic contracts, while Sinan owns real runtime semantics, project data, registry policy, localized content, input/focus/scroll/selection/text state, platform policy, fallback UI, and side effects.

- Debug: if a manifest validation fails, first check for stale paths or renamed tests.
- Architecture: every fixture names its owner and source-of-truth boundary.
- Scope: this document does not add datasource loading, project JSON mutation, production Canvas2D, real Sinan integration, or editor replacement.
